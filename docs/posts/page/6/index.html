Calisthenics

I didn't know about object calisthenics. They are a set of rules to help develop better: 1. One level of indentation per method2. Don’t use the ELSE keyword 3. Wrap all primitives and Strings4. First class collections5. One dot per line6. Don’t abbreviate7. Keep all entities small8. No classes with more than two instance variables9. No getters/setters/propertiesSome of them are a must, with some others I have some doubts. But the point 2 is so intriguing I have to test it.
